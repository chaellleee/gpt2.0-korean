# 개발 과정

강의에서 배운 GPT를 한국어 데이터로 학습시키는 과정에서 여러 문제를 만났고, 순서대로 고쳐 나갔다.
참고로 **Transformer 구조 자체는 처음부터 끝까지 바꾸지 않았다.** 강의의 GPT를 그대로 두고,
데이터·모델 크기·토크나이저·생성 방식만 상황에 맞게 조정했다.

## 1. 강의 설정 그대로 학습 → 과적합

처음에는 강의 최종 설정(384차원, 6층, 5000스텝, 문자 단위)을 그대로 써서 학습시켰다.
하지만 데이터가 강의의 1/4 수준이라, train loss는 계속 내려가는데 val loss는 step 1000 이후
다시 올라갔다. 모델이 데이터를 이해한 게 아니라 외워버린 것이다(과적합).

## 2. 데이터 2배로 늘림 → 과적합 완화

위키문헌에서 공개 작품을 더 모아 19편(28만 자)에서 39편(52만 자)으로 늘렸다.
그 결과 최저 val loss가 3.34에서 3.16으로 좋아졌고, 과적합이 시작되는 지점도 늦춰졌다.
데이터를 늘리는 것이 과적합의 근본적인 해결책임을 확인했다.

## 3. 모델을 줄임 → 이번엔 과소적합

데이터를 더 늘리기 어려워서, 반대로 모델을 작게 줄이고 dropout·weight decay를 강하게 걸었다.
과적합은 잡혔지만, 모델이 너무 약해져서 생성 결과가 같은 글자만 반복(`가가가`)하는 수준으로
무너졌다. 너무 줄이면 학습 자체를 못 한다는 걸 알게 됐다.

## 4. BPE 토크나이저를 직접 구현 → 글자 깨짐

문자 단위의 비효율을 줄이려고 BPE 토크나이저를 직접 만들었다. 처음에는 바이트 단위로 짰는데,
한글이 3바이트라 글자가 중간에서 잘려 깨진 문자(�)가 나왔다. 그래서 바이트 대신 문자에서
시작하도록 고쳤더니 깨짐이 사라졌다.

## 5. 최종 정리 → 균형점 찾기

여러 시도 끝에, 출력 품질이 가장 좋은 조합으로 정했다.

- 문자 단위 토크나이저 (깨짐 없음)
- 중간 크기 모델 (과적합과 과소적합의 사이)
- best checkpoint: val이 가장 낮았던 시점의 모델로 생성
- top-k 샘플링: 다음 글자를 그럴듯한 후보 중에서만 뽑아 문장을 자연스럽게

이 설정에서 대화체와 "…하였다" 문체가 살아있는, 읽을 만한 한국 근대문학풍 텍스트를 얻었다.

## 정리

과적합과 과소적합 사이에서 균형점을 찾는 과정이었다. 데이터를 늘리고, 모델 크기를 조절하고,
토크나이저를 고치고, 생성 방식을 다듬으면서 점점 나은 결과로 수렴했다. 핵심은
"작은 데이터에서는 모델을 무작정 키우거나 줄이는 게 아니라, 데이터 크기에 맞는 균형이 필요하다"는 점이다.
