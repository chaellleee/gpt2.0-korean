# 개발 과정 (DEVLOG) — 무엇이, 왜 바뀌었나

이 프로젝트는 한 번에 완성되지 않았습니다. **모델 구조(Transformer 본체)는 처음부터 끝까지
강의와 동일하게 유지**했지만, 작은 데이터에서 좋은 결과를 얻기까지 **모델 크기 · 토크나이저 ·
정규화 · 생성 방식**을 여러 번 바꿔야 했습니다. 그 과정을 기록합니다.

> 핵심 오해 방지: "Transformer가 바뀌었다"기보다, **Transformer는 그대로 두고 그 주변 설정을
> 데이터 상황에 맞춰 조정**한 것입니다. 아래의 모든 변경은 같은 GPT 구조 위에서 이루어졌습니다.

---

## 1단계 — 강의 그대로 (full GPT, char-level)

처음엔 강의 최종 설정을 그대로 썼습니다: `n_embd=384, n_layer=6, n_head=6, block_size=256,
dropout=0.2, 5000 steps`, 문자 단위 토크나이저. 약 1000만 파라미터.

**문제: 심각한 과적합(overfitting).**
데이터가 강의(110만 자)의 1/4 수준(28만 자)이라, train loss는 계속 내려가는데 val loss는
step 1000 부근에서 최저(3.34)를 찍고 **그 뒤로 폭주(→ 5.37)** 했습니다. 모델이 데이터를
"이해"한 게 아니라 "통째로 암기"한 것입니다.

→ **교훈:** 모델 용량에 비해 데이터가 너무 작으면 외워버린다.

## 2단계 — 데이터 2배 확장 (28만 → 52만 자)

위키문헌에서 퍼블릭 도메인 작품을 더 수집해 19편 → 39편으로 늘렸습니다.

**결과: 과적합이 완화됨.** 최저 val loss 3.34 → **3.16**으로 개선, 과적합 시작 지점도
step 1000 → **1750**으로 늦춰지고, 그 후 상승 폭도 훨씬 완만해졌습니다.

→ **교훈:** 과적합의 가장 근본적인 해법은 데이터를 늘리는 것. (단, 위키문헌의 공개 단편으로는
52만 자가 현실적 상한이었음)

## 3단계 — 모델 축소 + 강한 정규화 (Micro-GPT)

데이터를 더 못 늘리니, 이번엔 모델을 줄여 과적합을 막기로 했습니다:
`n_embd=128, n_layer=3, block 64`, `dropout 0.3`, `weight_decay 0.1`, `weight tying`,
`early stopping`.

**문제: 이번엔 반대로 과소적합(underfitting).**
과적합은 잡혔지만 모델이 너무 약해져서, 생성 결과가 흔한 조사만 반복(`가가가`, `그그그`)하는
수준으로 무너졌습니다.

→ **교훈:** 과적합과 과소적합은 양 끝. 너무 줄이면 학습 자체를 못 한다. **균형점**이 필요하다.

## 4단계 — 토크나이저 실험: 직접 짠 BPE

문자 단위(어휘 1,908)의 비효율을 극복하려고, 라이브러리 없이 **BPE 토크나이저를 직접 구현**했습니다.

**문제: 깨진 글자(`�`) 발생.**
처음엔 UTF-8 **바이트** 단위로 BPE를 짰는데, 한글 한 글자가 3바이트라 모델이 글자 중간에서
토큰을 잘라 뱉으면 유효하지 않은 바이트가 되어 `�`로 깨졌습니다.

**해결: BPE의 기반을 "바이트" → "문자"로 변경.**
고유 문자에서 시작해 병합하니, 토큰이 항상 완전한 글자들의 묶음이 되어 **깨짐이 구조적으로
불가능**해졌습니다.

→ **교훈:** 한국어처럼 멀티바이트 문자는 토크나이저 설계가 출력 품질에 직접 영향을 준다.

## 5단계 — 최종 결정: 중간 크기 char-level + 생성 기법

여러 시도 끝에, **출력 품질** 관점에서 가장 좋은 조합으로 확정했습니다:

- **문자 단위 토크나이저** (깨짐 없음, 강의에 충실)
- **중간 크기 모델** (`n_embd=256, n_layer=4` — full과 Micro의 사이, 균형점)
- **best checkpoint** — val이 가장 낮았던(=과적합 전) 시점의 모델을 저장해 생성에 사용
- **top-k + temperature 샘플링** — 다음 글자를 그럴듯한 후보 중에서만 뽑아 문장이 자연스러워짐

이 조합에서, 대화체와 "…하였다" 문체가 살아있는 **읽을 만한 한국 근대문학풍 텍스트**가 나왔습니다.

## 부수적으로 겪은 문제들

- **노트북 셀 순서 꼬임** — 모델 정의 셀을 다시 안 돌리고 학습/생성 셀만 실행해
  `NameError: model is not defined`나 이전 가중치 재사용 같은 오류가 반복됨.
  → **모든 과정을 한 셀에 합친 `GPT_korean_ONECELL.ipynb`로 해결.**
- **가중치 초기화 누락** — 초기 손실이 `ln(vocab)≈7.9`가 아니라 42에서 시작.
  표준 초기화(`std=0.02`)를 추가해 정상화.

## 한 줄 요약

| 단계 | 바꾼 것 | 이유 |
|---|---|---|
| 1 | 강의 그대로 | 출발점 |
| 2 | 데이터 2배 | 과적합 → 데이터 부족이 근본 원인 |
| 3 | 모델 축소·정규화 | 과적합 억제 (→ 과소적합 부작용) |
| 4 | BPE 직접 구현 → 문자기반 | 토큰 효율 시도 → 깨짐(�) 해결 |
| 5 | 중간 크기 + best ckpt + top-k | 과적합·과소적합의 균형 + 출력 품질 |

**Transformer 구조 자체는 1~5단계 내내 동일했습니다.** 바뀐 것은 데이터 양, 모델 크기,
토크나이저, 정규화, 생성 방식 — 모두 "작은 데이터로 좋은 한국어를 생성한다"는 목표를 위한 조정이었습니다.
